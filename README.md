# Сервис записи на курсы «ОчУмелые ручки»

PHP-приложение лабораторной работы №9. Реализует двухстороннюю валидацию (на фронте и на бэкенде), CSRF-защиту, хеширование паролей, проверку слотов мастер-классов и управление записями.

## Стек

- PHP 8.1+ (использует `readonly`-свойства, `never` return type)
- SQLite (PDO; БД создаётся автоматически)
- Vanilla JS на фронте

## Запуск

```bash
# 1. Создать и наполнить БД (один раз; повторный запуск пересоздаст её)
php scripts/init_db.php

# 2. Запустить встроенный PHP-сервер
php -S 127.0.0.1:8000 -t public
```

Откройте http://127.0.0.1:8000/

## Безопасность

- Все формы защищены CSRF-токеном (`<meta name="csrf-token">` + hidden поле).
- Пароли хешируются `password_hash()` (bcrypt).
- Все запросы к БД — подготовленные выражения PDO.
- В шаблонах — `htmlspecialchars()` через хелпер `e()`.
- Сессионная cookie: HTTPOnly + SameSite=Lax; `session_regenerate_id()` при логине.
- Заголовки: `X-Content-Type-Options: nosniff`, `X-Frame-Options: SAMEORIGIN`, `Referrer-Policy`.
- Валидация на двух уровнях:
  - **frontend** — `data-rule` атрибуты + `app.js` (Validators) + `required/min/max/pattern`;
  - **backend** — класс `Validator` с цепочечными правилами; ошибки кладутся во flash и отображаются у соответствующих полей.

## Реализованный функционал (по ТЗ)

### Расписание мастер-классов (для ведущего)
- Авторизация ведущего → редирект на `Личный кабинет`.
- Создание мастер-класса → запись в БД; только под учётной записью ведущего (роль `master`).
- Все поля (вид творчества, название, описание, дата, время, кол-во человек, стоимость) — обязательны; валидация на обеих сторонах.
- Занятые слоты текущего ведущего на выбранную дату подгружаются AJAX'ом (`?p=api_slots`) и помечаются как `disabled` в `<select>`. На бэкенде дополнительно — UNIQUE-индекс `(master_id, date, time_slot)` + явная проверка.
- Сетка занятий 9–11, 11–13, 13–15, 15–17ч — задана `CHECK`-ограничением в БД и `Validator::in()`.
- В кабинете отображаются только мастер-классы текущего ведущего; разделы `Участники` и `Редактировать`.
- Редактирование разрешено только для описания и стоимости (остальные поля выведены, но `readonly`/`disabled`; контроллер обновляет только `description` и `price`).

### Запись на мастер-класс (для посетителя)
- Главная страница до авторизации показывает виды творчества и ссылки `Вход / Регистрация`.
- Кнопка `записаться` видна только авторизованному пользователю (иначе выводится «Войдите для записи»).
- Регистрация: проверка email (FILTER_VALIDATE_EMAIL + длина) и телефона (`+7XXXXXXXXXX`/`8…`).
- Подтверждение записи — отдельная страница со сводкой (ФИО, вид творчества, ведущий, дата, время) и кнопками `Подтвердить` / `Отменить`. После — флеш-сообщение и редирект на страницу вида творчества.
- Авторизованный пользователь видит свои записи на главной, может отменить.
- Запрет двойной записи (`UNIQUE(user_id, master_class_id)` + предварительная проверка).
- Запрет записи при отсутствии свободных мест (проверка в транзакции).
- Запрет записи мастера на собственный класс.
